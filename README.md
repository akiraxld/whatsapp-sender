# WhatsApp Sender

Программа читает таблицу с номерами и сообщениями, и либо симулирует отправку (dry-run), либо отправляет сообщение на тестовый номер через WhatsApp Web (live).

---

## Установка

**Требования:** Python 3.10+, Google Chrome

1. Клонировать репозиторий:
   ```bash
   git clone https://github.com/akiraxld/whatsapp-sender.git
   cd whatsapp-sender
   ```

3. Установить зависимости:
   ```bash
    pip install selenium pandas
    ```

## Запуск

### Режим dry-run (по умолчанию)
Сообщения **не отправляются** — только пишутся в лог.

```bash
python src/main.py
```

или явно:

```bash
python src/main.py dry-run
```

Лог сохраняется в `logs/dry_run.log`

---

### Режим live
Отправляет сообщение на тестовый номер через WhatsApp Web.

```bash
python src/main.py live
```

1. Откроется браузер Chrome с WhatsApp Web
2. Отсканируй QR-код своим телефоном
3. Программа сама найдёт тестовый номер, подождёт случайную паузу и отправит сообщение

## Обработка ошибок

Программа корректно обрабатывает и логирует следующие кейсы:

| Кейс | Пример | Что происходит |
|---|---|---|
| Пустая строка / NaN | нет номера | `[ERROR] Строка N: Номер телефона отсутствует` |
| Текст вместо номера | `abcdef` | `[ERROR] Строка N: содержит недопустимые символы` |
| Короткий / длинный номер | `+7707` | `[ERROR] Строка N: неверная длина` |
| Номер через 8 | `87071234567` | автоматически исправляется на `+7...` |
| Лишние символы | `+7 (707) 123-45-67` | автоматически очищается |
| Номер не в WhatsApp | валидный но не зарегистрирован | `[FAILED] Номер не зарегистрирован в WhatsApp` |
| Нет интернета | обрыв соединения | `[ERROR] Нет интернет-соединения. Ждём 30 секунд...` |
| Лимиты WhatsApp | слишком быстро | случайная пауза 10–25 сек между отправками |
